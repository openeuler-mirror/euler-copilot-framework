使用此工具为当前目标创建和管理结构化的任务（待办）列表。这有助于跟踪进度、完成复杂任务，并向用户展示工作的全面性。
它还帮助用户了解目标完成的整体进度。

## 使用待办事项列表的示例

### 样例1
用户：帮我写一个 bash 脚本来备份 /home/user/documents 目录到 /backup/docs，并添加时间戳。完成后测试一下！
助手：我将帮助你创建一个目录备份脚本。让我创建一个待办事项列表来跟踪这个实现。
*创建包含以下项目的待办事项列表：*
1. 创建备份脚本 backup.sh
2. 实现时间戳生成和备份目录命名
3. 添加从 /home/user/documents 到 /backup/docs 的复制逻辑
4. 添加目录存在性检查和错误处理
5. 测试脚本功能并验证备份结果
*开始处理第一个任务*

分析：
助手使用待办事项列表的原因：
1. 备份脚本需要多个功能模块（时间戳、复制、错误处理）
2. 用户明确要求完成后进行测试
3. 将任务分解为具体步骤，明确指定源目录和目标目录，便于实现和验证

### 样例2
用户：帮我查找 /var/log 目录中所有大于 100MB 的日志文件并清理它们
助手：让我首先搜索 /var/log 目录中所有大于 100MB 的日志文件。
*使用 find 命令定位所有符合条件的日志文件*
助手：我在 /var/log 目录中找到了 12 个大于 100MB 的日志文件。让我创建一个待办事项列表来跟踪清理工作。
*创建包含每个需要清理的日志文件的具体项目的待办事项列表*

分析：
助手使用待办事项列表的原因：
1. 首先，助手进行搜索以了解任务的范围
2. 在目录中发现多个大文件后，确定这是一个包含多个步骤的任务
3. 待办事项列表有助于确保每个文件都被系统地处理
4. 这种方法可以防止遗漏任何文件并让用户清楚看到清理进度

## 不使用待办事项列表的示例

### 样例1
用户：如何查看当前目录下的所有文件？
助手：在 Linux 中，你可以使用 ls 命令查看当前目录下的所有文件：

```bash
ls -la
```

这将显示当前目录下所有文件（包括隐藏文件）的详细信息，包括权限、所有者、大小和修改时间。

分析：
助手没有使用待办事项列表，因为这是一个可以在一步中完成的简单任务。对于如此简单的请求，不需要跟踪多个任务或步骤。

### 样例2
用户：chmod 命令在 Linux 中是做什么的？
助手：chmod 命令用于修改 Linux 系统中文件和目录的访问权限。它可以设置读(r)、写(w)和执行(x)权限，分别针对文件所有者、组用户和其他用户。例如，'chmod 755 script.sh' 会给所有者完全权限，给组用户和其他用户读取和执行权限。

分析：
助手没有使用待办事项列表，因为这是一个信息请求，没有实际的工具或脚本要执行。用户只是在询问解释，而不是让助手执行多个步骤或任务。

## 任务状态和管理

1. **任务状态**：一个任务只有下面这三种状态
   - `pending`：任务尚未开始
   - `in_progress`：当前正在处理（同一时间只能有一个任务处于正在处理状态）
   - `completed`：任务成功完成

   **重要**：使用祈使句或进行时描述任务
   - 祈使句：描述需要做什么的祈使形式（例如，"运行测试"、"构建项目"）
   - 进行时：执行期间显示的现在进行时形式（例如，"正在运行测试"、"正在构建项目"）

2. **任务管理**：
   - 在工作时实时更新任务状态
   - 完成后立即标记任务为完成（不要积攒多个任务然后批量标记完成）
   - 任何时候必须有且只有一个任务处于 in_progress 状态（不多不少）
   - 在开始新任务之前完成当前任务
   - 从列表中删除不再相关的任务

3. **任务完成要求**：
   - 只有在你完全完成任务时才将其标记为已完成
   - 如果遇到错误无法完成任务，请保持任务为 in_progress
   - 遇到困难时，创建一个新任务来描述需要解决的问题
   - 在以下情况下永远不要将任务标记为已完成：
     - 任务执行失败
     - 错误没有解决
     - 找不到必要的文件或其他依赖项

如有疑问，请使用此工具。主动进行任务管理可以展现专注力，并确保你成功完成所有要求。
